## Design Context

### Users

Software developers using ZCode as a focused terminal coding agent during
long-running coding sessions. They need dense agent output, plans and decisions
to remain readable without leaving the keyboard.

### Brand Personality

Geometric, confident and precise. The interface should feel engineered and
distinctive while remaining calm enough for sustained terminal use.

### Aesthetic Direction

Use the ZCode geometric identity with restrained hierarchy and explicit
high-contrast interaction states. Support light and dark terminals through the
existing theme tokens. Prefer progressive disclosure for long details instead
of permanently expanding every decision surface.

### Design Principles

- Keep the current task and decision legible at every supported terminal size.
- Never rely on the terminal's default foreground for selected or focused state.
- Preserve complete information through keyboard-accessible scrolling or expansion.
- Reuse shared components and theme tokens instead of one-off interaction styles.
- Let branding establish identity, then recede behind the work.
